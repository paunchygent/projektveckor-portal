# Foundational principles

## MUST

- Keep the portal teacher-first: clarity > cleverness.
- Prefer stable URLs. Content should be linkable for years.
- Use external CSS (no inline styles) for any teacher-facing HTML we may export/reuse.
- Keep deployment repeatable: Compose + environment variables.

## SHOULD

- Minimize moving parts: start as a static SPA with a thin backend for health checks + static hosting.
- Write docs as the source of truth for workflows.
